Box-Behnken Design

Parking Lot Layout Design

Box-Behnken design to maximize capacity and minimize average walking distance by tuning stall angle, aisle width, and stall width

Summary

This experiment investigates parking lot layout design. Box-Behnken design to maximize capacity and minimize average walking distance by tuning stall angle, aisle width, and stall width.

The design varies 3 factors: stall angle (deg), ranging from 45 to 90, aisle width (m), ranging from 4.5 to 7.5, and stall width (m), ranging from 2.4 to 3.0. The goal is to optimize 2 responses: capacity (spaces) (maximize) and avg walk m (m) (minimize). Fixed conditions held constant across all runs include lot area m2 = 5000, handicap pct = 5.

A Box-Behnken design was chosen because it efficiently fits quadratic models with 3 continuous factors while avoiding extreme corner combinations — requiring only 15 runs instead of the 8 needed for a full factorial at two levels.

Quadratic response surface models were fitted to capture potential curvature and factor interactions. The RSM contour plots below visualize how pairs of factors jointly affect each response.

Key Findings

For capacity, the most influential factors were stall angle (46.2%), stall width (39.0%), aisle width (14.8%). The best observed value was 225.0 (at stall angle = 67.5, aisle width = 7.5, stall width = 2.4).

For avg walk m, the most influential factors were stall width (58.0%), stall angle (24.5%), aisle width (17.5%). The best observed value was 35.0 (at stall angle = 67.5, aisle width = 7.5, stall width = 3).

Recommended Next Steps

  • Run confirmation experiments at the predicted optimal settings to validate the model.
  • Consider whether any fixed factors should be varied in a future study.

Step-by-Step Workflow

1

Preview the design

Terminal
$ python doe.py info --config use_cases/125_parking_lot_layout/config.json
2

Generate the runner script

Terminal
$ python doe.py generate --config use_cases/125_parking_lot_layout/config.json \ --output use_cases/125_parking_lot_layout/results/run.sh --seed 42
3

Execute the experiments

Terminal
$ bash use_cases/125_parking_lot_layout/results/run.sh
4

Analyze results

Terminal
$ python doe.py analyze --config use_cases/125_parking_lot_layout/config.json
5

Get optimization recommendations

Terminal
$ python doe.py optimize --config use_cases/125_parking_lot_layout/config.json
6

Multi-objective optimization

With 2 competing responses, use --multi to find the best compromise via Derringer–Suich desirability.

Terminal
$ python doe.py optimize --config use_cases/125_parking_lot_layout/config.json --multi
7

Generate the HTML report

Terminal
$ python doe.py report --config use_cases/125_parking_lot_layout/config.json \ --output use_cases/125_parking_lot_layout/results/report.html
