Plackett-Burman Design

DDoS Mitigation Threshold

Plackett-Burman screening of 6 anti-DDoS parameters for detection accuracy and clean traffic impact

Summary

This experiment investigates ddos mitigation threshold. Plackett-Burman screening of 6 anti-DDoS parameters for detection accuracy and clean traffic impact.

The design varies 6 factors: syn rate limit (pps), ranging from 1000 to 50000, connection limit (conns/IP), ranging from 100 to 10000, challenge mode, ranging from off to javascript, geo blocking, ranging from off to on, anomaly window s (s), ranging from 10 to 300, and whitelist size (IPs), ranging from 100 to 10000. The goal is to optimize 2 responses: detection rate (%) (maximize) and false positive pct (%) (minimize). Fixed conditions held constant across all runs include mitigation provider = in_house, baseline traffic = 5000_rps.

A Plackett-Burman screening design was used to efficiently test 6 factors in only 8 runs. This design assumes interactions are negligible and focuses on identifying the most influential main effects.

Key Findings

For detection rate, the most influential factors were geo blocking (33.6%), connection limit (23.3%), anomaly window s (20.8%). The best observed value was 101.9 (at syn rate limit = 1000, connection limit = 100, challenge mode = javascript).

For false positive pct, the most influential factors were anomaly window s (28.2%), whitelist size (26.1%), geo blocking (20.4%). The best observed value was -0.24 (at syn rate limit = 1000, connection limit = 100, challenge mode = javascript).

Recommended Next Steps

  • Follow up with a response surface design (CCD or Box-Behnken) on the top 3–4 factors to model curvature and find the true optimum.
  • Consider whether any fixed factors should be varied in a future study.
  • The screening results can guide factor reduction — drop factors contributing less than 5% and re-run with a smaller, more focused design.

Step-by-Step Workflow

1

Preview the design

Terminal
$ python doe.py info --config use_cases/53_ddos_mitigation_threshold/config.json
2

Generate the runner script

Terminal
$ python doe.py generate --config use_cases/53_ddos_mitigation_threshold/config.json \ --output use_cases/53_ddos_mitigation_threshold/results/run.sh --seed 42
3

Execute the experiments

Terminal
$ bash use_cases/53_ddos_mitigation_threshold/results/run.sh
4

Analyze results

Terminal
$ python doe.py analyze --config use_cases/53_ddos_mitigation_threshold/config.json
5

Get optimization recommendations

Terminal
$ python doe.py optimize --config use_cases/53_ddos_mitigation_threshold/config.json
6

Multi-objective optimization

With 2 competing responses, use --multi to find the best compromise via Derringer–Suich desirability.

Terminal
$ python doe.py optimize --config use_cases/53_ddos_mitigation_threshold/config.json --multi
7

Generate the HTML report

Terminal
$ python doe.py report --config use_cases/53_ddos_mitigation_threshold/config.json \ --output use_cases/53_ddos_mitigation_threshold/results/report.html
