Reptile Terrarium Setup

Full factorial of basking temperature, humidity, UVB output, and substrate depth to maximize activity level and minimize stress indicators

Summary

This experiment investigates reptile terrarium setup. Full factorial of basking temperature, humidity, UVB output, and substrate depth to maximize activity level and minimize stress indicators.

The design varies 4 factors: basking c (C), ranging from 30 to 40, humidity pct (%), ranging from 30 to 70, uvb pct (%UVI), ranging from 5 to 14, and substrate cm (cm), ranging from 3 to 15. The goal is to optimize 2 responses: activity score (pts) (maximize) and stress indicators (pts) (minimize). Fixed conditions held constant across all runs include species = bearded_dragon, enclosure L = 120cm.

A full factorial design was used to explore all 16 possible combinations of the 4 factors at two levels. This guarantees that every main effect and interaction can be estimated independently, at the cost of a larger experiment (16 runs).

Quadratic response surface models were fitted to capture potential curvature and factor interactions. The RSM contour plots below visualize how pairs of factors jointly affect each response.

Key Findings

For activity score, the most influential factors were humidity pct (47.9%), basking c (31.3%), substrate cm (13.2%). The best observed value was 6.7 (at basking c = 30, humidity pct = 70, uvb pct = 5).

For stress indicators, the most influential factors were basking c (71.3%), substrate cm (21.3%), humidity pct (4.6%). The best observed value was 3.0 (at basking c = 40, humidity pct = 30, uvb pct = 5).

Recommended Next Steps

  • Consider whether any fixed factors should be varied in a future study.

Step-by-Step Workflow

1

Preview the design

Terminal
$ python doe.py info --config use_cases/174_reptile_habitat/config.json
2

Generate the runner script

Terminal
$ python doe.py generate --config use_cases/174_reptile_habitat/config.json \ --output use_cases/174_reptile_habitat/results/run.sh --seed 42
3

Execute the experiments

Terminal
$ bash use_cases/174_reptile_habitat/results/run.sh
4

Analyze results

Terminal
$ python doe.py analyze --config use_cases/174_reptile_habitat/config.json
5

Get optimization recommendations

Terminal
$ python doe.py optimize --config use_cases/174_reptile_habitat/config.json

Multi-Objective Optimization

Terminal
$ python doe.py optimize --config use_cases/174_reptile_habitat/config.json --multi
6

Generate the HTML report

Terminal
$ python doe.py report --config use_cases/174_reptile_habitat/config.json \ --output use_cases/174_reptile_habitat/results/report.html

Real-World Experiment Workflow

ℹ

Physical Animal Care Experiment? Use the Manual Workflow

Animal experiments involve living creatures with variable behavior, biological needs, and environmental responses that can't be simulated. The simulation script above is for demonstration. For your real experiments, use record, status, and export-worksheet.

Manual experiment workflow
# 1. Print a worksheet for the lab / field / kitchen $ python doe.py export-worksheet --config use_cases/174_reptile_habitat/config.json \ --format csv --output worksheet.csv # 2. Check which runs to do next $ python doe.py status --config use_cases/174_reptile_habitat/config.json # 3. After each experiment, record the measurements $ python doe.py record --config use_cases/174_reptile_habitat/config.json --run 1 # 4. Peek at partial results as you go $ python doe.py analyze --config use_cases/174_reptile_habitat/config.json --partial # 5. Full analysis when all runs are complete $ python doe.py analyze --config use_cases/174_reptile_habitat/config.json $ python doe.py optimize --config use_cases/174_reptile_habitat/config.json $ python doe.py report --config use_cases/174_reptile_habitat/config.json \ --output report.html
✔

Works Without a Test Script

The analysis pipeline doesn't care how results were produced. Record your measurements with record, and the full analysis, optimization, and reporting pipeline works identically. Use --partial to get early insights before all runs are complete.
