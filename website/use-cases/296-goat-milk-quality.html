Box-Behnken Design

Goat Milk Quality Factors

Box-Behnken design to maximize butterfat content and minimize somatic cell count by tuning concentrate supplementation, milking frequency, and pasture access hours

Summary

This experiment investigates goat milk quality factors. Box-Behnken design to maximize butterfat content and minimize somatic cell count by tuning concentrate supplementation, milking frequency, and pasture access hours.

The design varies 3 factors: concentrate g (g/day), ranging from 200 to 800, milking freq (per_day), ranging from 1 to 3, and pasture hrs (hrs), ranging from 4 to 12. The goal is to optimize 2 responses: butterfat pct (%) (maximize) and scc k ml (k/mL) (minimize). Fixed conditions held constant across all runs include breed = saanen, lactation week = 12.

A Box-Behnken design was chosen because it efficiently fits quadratic models with 3 continuous factors while avoiding extreme corner combinations — requiring only 15 runs instead of the 8 needed for a full factorial at two levels.

Quadratic response surface models were fitted to capture potential curvature and factor interactions. The RSM contour plots below visualize how pairs of factors jointly affect each response.

Key Findings

For butterfat pct, the most influential factors were concentrate g (67.0%), pasture hrs (24.8%), milking freq (8.2%). The best observed value was 3.7 (at concentrate g = 200, milking freq = 1, pasture hrs = 8).

For scc k ml, the most influential factors were concentrate g (60.0%), milking freq (24.2%), pasture hrs (15.8%). The best observed value was 133.0 (at concentrate g = 500, milking freq = 2, pasture hrs = 8).

Recommended Next Steps

  • Run confirmation experiments at the predicted optimal settings to validate the model.
  • Consider whether any fixed factors should be varied in a future study.

Step-by-Step Workflow

1

Preview the design

Terminal

2

Generate the runner script

Terminal

3

Execute the experiments

Terminal
$ bash use_cases/296_goat_milk_quality/results/run.sh

Analyze results

Terminal

Real-World Experiment Workflow

Physical Farm & Livestock Experiment? Use the Manual Workflow

Livestock experiments involve real animals, feed, facilities, and biological responses that can't be simulated. The simulation script above is for demonstration. For your real experiments, use record, status, and export-worksheet.

Manual experiment workflow
# 1. Print a worksheet for the lab / field / kitchen $ python doe.py export-worksheet --config use_cases/296_goat_milk_quality/config.json \ --format csv --output worksheet.csv # 2. Check which runs to do next $ python doe.py status --config use_cases/296_goat_milk_quality/config.json # 3. After each experiment, record the measurements $ python doe.py record --config use_cases/296_goat_milk_quality/config.json --run 1 # 4. Peek at partial results as you go $ python doe.py analyze --config use_cases/296_goat_milk_quality/config.json --partial # 5. Full analysis when all runs are complete $ python doe.py analyze --config use_cases/296_goat_milk_quality/config.json $ python doe.py optimize --config use_cases/296_goat_milk_quality/config.json $ python doe.py report --config use_cases/296_goat_milk_quality/config.json \ --output report.html
✔

Works Without a Test Script

The analysis pipeline doesn't care how results were produced. Record your measurements with record, and the full analysis, optimization, and reporting pipeline works identically. Use --partial to get early insights before all runs are complete.
