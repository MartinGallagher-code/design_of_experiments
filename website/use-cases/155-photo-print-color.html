Box-Behnken Design

Photo Print Color Accuracy

Box-Behnken design to maximize color accuracy and minimize ink cost by tuning color profile, ink density, and paper type brightness

Summary

This experiment investigates photo print color accuracy. Box-Behnken design to maximize color accuracy and minimize ink cost by tuning color profile, ink density, and paper type brightness.

The design varies 3 factors: profile gamma (gamma), ranging from 1.8 to 2.4, ink density pct (%), ranging from 80 to 120, and paper brightness (ISO), ranging from 90 to 100. The goal is to optimize 2 responses: delta e (dE) (minimize) and ink ml (mL/m2) (minimize). Fixed conditions held constant across all runs include printer = inkjet_8color, resolution dpi = 1440.

A Box-Behnken design was chosen because it efficiently fits quadratic models with 3 continuous factors while avoiding extreme corner combinations — requiring only 15 runs instead of the 8 needed for a full factorial at two levels.

Quadratic response surface models were fitted to capture potential curvature and factor interactions. The RSM contour plots below visualize how pairs of factors jointly affect each response.

Key Findings

For delta e, the most influential factors were paper brightness (49.8%), profile gamma (35.1%), ink density pct (15.0%). The best observed value was 3.8 (at profile gamma = 1.8, ink density pct = 100, paper brightness = 90).

For ink ml, the most influential factors were profile gamma (51.9%), paper brightness (46.6%), ink density pct (1.5%). The best observed value was 5.6 (at profile gamma = 1.8, ink density pct = 120, paper brightness = 95).

Recommended Next Steps

  • Run confirmation experiments at the predicted optimal settings to validate the model.
  • Consider whether any fixed factors should be varied in a future study.

Real-World Experiment Workflow

ℹ

Physical Photography Test? Use the Manual Workflow

Photography experiments involve real light, optics, and exposure conditions. Results depend on physical phenomena that must be captured and evaluated. The simulation script above is for demonstration. For your real experiments, use record, status, and export-worksheet.

Manual experiment workflow
# 1. Print a worksheet for the lab / field / kitchen $ python doe.py export-worksheet --config use_cases/155_photo_print_color/config.json \ --format csv --output worksheet.csv # 2. Check which runs to do next $ python doe.py status --config use_cases/155_photo_print_color/config.json # 3. After each experiment, record the measurements $ python doe.py record --config use_cases/155_photo_print_color/config.json --run 1 # 4. Peek at partial results as you go $ python doe.py analyze --config use_cases/155_photo_print_color/config.json --partial # 5. Full analysis when all runs are complete $ python doe.py analyze --config use_cases/155_photo_print_color/config.json $ python doe.py optimize --config use_cases/155_photo_print_color/config.json $ python doe.py report --config use_cases/155_photo_print_color/config.json \ --output report.html
✔

Works Without a Test Script

The analysis pipeline doesn't care how results were produced. Record your measurements with record, and the full analysis, optimization, and reporting pipeline works identically. Use --partial to get early insights before all runs are complete.
