Box-Behnken Design

pH Buffer Preparation

Box-Behnken design to maximize buffer capacity and minimize temperature sensitivity by tuning buffer concentration, acid-base ratio, and ionic strength

Summary

This experiment investigates ph buffer preparation. Box-Behnken design to maximize buffer capacity and minimize temperature sensitivity by tuning buffer concentration, acid-base ratio, and ionic strength.

The design varies 3 factors: concentration mm (mM), ranging from 10 to 100, acid base ratio (ratio), ranging from 0.5 to 2.0, and ionic strength mm (mM), ranging from 50 to 300. The goal is to optimize 2 responses: buffer capacity (mmol/L/pH) (maximize) and temp sensitivity (dpH/dC) (minimize). Fixed conditions held constant across all runs include buffer system = phosphate, target ph = 7.4.

A Box-Behnken design was chosen because it efficiently fits quadratic models with 3 continuous factors while avoiding extreme corner combinations — requiring only 15 runs instead of the 8 needed for a full factorial at two levels.

Quadratic response surface models were fitted to capture potential curvature and factor interactions. The RSM contour plots below visualize how pairs of factors jointly affect each response.

Key Findings

For buffer capacity, the most influential factors were concentration mm (49.2%), ionic strength mm (28.4%), acid base ratio (22.4%). The best observed value was 21.5 (at concentration mm = 10, acid base ratio = 1.25, ionic strength mm = 300).

For temp sensitivity, the most influential factors were acid base ratio (44.4%), ionic strength mm (37.1%), concentration mm (18.5%). The best observed value was 0.0025 (at concentration mm = 100, acid base ratio = 1.25, ionic strength mm = 300).

Recommended Next Steps

  • Run confirmation experiments at the predicted optimal settings to validate the model.
  • Consider whether any fixed factors should be varied in a future study.

Step-by-Step Workflow

1

Preview the design

Terminal
$ python doe.py info --config use_cases/191_ph_buffer_prep/config.json
2

Generate the runner script

Terminal
$ python doe.py generate --config use_cases/191_ph_buffer_prep/config.json \ --output use_cases/191_ph_buffer_prep/results/run.sh --seed 42
3

Execute the experiments

Terminal
$ bash use_cases/191_ph_buffer_prep/results/run.sh
4

Analyze results

Terminal
$ python doe.py analyze --config use_cases/191_ph_buffer_prep/config.json
5

Get optimization recommendations

Terminal
$ python doe.py optimize --config use_cases/191_ph_buffer_prep/config.json
6

Multi-objective optimization

With 2 competing responses, use --multi to find the best compromise via Derringer–Suich desirability.

Terminal
$ python doe.py optimize --config use_cases/191_ph_buffer_prep/config.json --multi
7

Generate the HTML report

Terminal
$ python doe.py report --config use_cases/191_ph_buffer_prep/config.json \ --output use_cases/191_ph_buffer_prep/results/report.html
