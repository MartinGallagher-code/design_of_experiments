Central Composite Design

Rowing Ergometer Performance

Central composite design to maximize power output and minimize split time by tuning stroke rate, drive ratio, and drag factor

Summary

This experiment investigates rowing ergometer performance. Central composite design to maximize power output and minimize split time by tuning stroke rate, drive ratio, and drag factor.

The design varies 3 factors: stroke rate (spm), ranging from 22 to 34, drive ratio (ratio), ranging from 1.5 to 3.0, and drag factor (units), ranging from 100 to 140. The goal is to optimize 2 responses: avg watts (W) (maximize) and split 500m sec (sec) (minimize). Fixed conditions held constant across all runs include athlete = intermediate_male, piece = 2000m.

A Central Composite Design (CCD) was selected to fit a full quadratic response surface model, including curvature and interaction effects. With 3 factors this produces 22 runs including center points and axial (star) points that extend beyond the factorial range.

Quadratic response surface models were fitted to capture potential curvature and factor interactions. The RSM contour plots below visualize how pairs of factors jointly affect each response.

Key Findings

For avg watts, the most influential factors were drive ratio (39.2%), drag factor (31.3%), stroke rate (29.5%). The best observed value was 242.0 (at stroke rate = 28, drive ratio = 2.25, drag factor = 83.4852).

For split 500m sec, the most influential factors were drive ratio (40.4%), stroke rate (29.8%), drag factor (29.8%). The best observed value was 108.0 (at stroke rate = 28, drive ratio = 2.25, drag factor = 83.4852).

Recommended Next Steps

  • Run confirmation experiments at the predicted optimal settings to validate the model.
  • Consider whether any fixed factors should be varied in a future study.

Step-by-Step Workflow

1

Preview the design

Terminal
$ python doe.py info --config use_cases/216_rowing_technique/config.json
2

Generate the runner script

Terminal
$ python doe.py generate --config use_cases/216_rowing_technique/config.json \ --output use_cases/216_rowing_technique/results/run.sh --seed 42
3

Execute the experiments

Terminal
$ bash use_cases/216_rowing_technique/results/run.sh
4

Analyze results

Terminal
$ python doe.py analyze --config use_cases/216_rowing_technique/config.json
5

Get optimization recommendations

Terminal
$ python doe.py optimize --config use_cases/216_rowing_technique/config.json
6

Multi-objective optimization

With 2 competing responses, use --multi to find the best compromise via Derringer–Suich desirability.

Terminal
$ python doe.py optimize --config use_cases/216_rowing_technique/config.json --multi
7

Generate the HTML report

Terminal
$ python doe.py report --config use_cases/216_rowing_technique/config.json \ --output use_cases/216_rowing_technique/results/report.html
