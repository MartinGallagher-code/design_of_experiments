Box-Behnken Design

Fossil Preparation Technique

Box-Behnken design to maximize fossil detail preservation and minimize matrix damage by tuning air abrasive pressure, nozzle distance, and media grit

Summary

This experiment investigates fossil preparation technique. Box-Behnken design to maximize fossil detail preservation and minimize matrix damage by tuning air abrasive pressure, nozzle distance, and media grit.

The design varies 3 factors: pressure psi (psi), ranging from 20 to 80, nozzle mm (mm), ranging from 5 to 30, and media mesh (mesh), ranging from 100 to 400. The goal is to optimize 2 responses: detail score (pts) (maximize) and damage score (pts) (minimize). Fixed conditions held constant across all runs include media type = sodium_bicarbonate, fossil = trilobite.

A Box-Behnken design was chosen because it efficiently fits quadratic models with 3 continuous factors while avoiding extreme corner combinations — requiring only 15 runs instead of the 8 needed for a full factorial at two levels.

Quadratic response surface models were fitted to capture potential curvature and factor interactions. The RSM contour plots below visualize how pairs of factors jointly affect each response.

Key Findings

For detail score, the most influential factors were media mesh (42.8%), pressure psi (35.8%), nozzle mm (21.4%). The best observed value was 6.8 (at pressure psi = 50, nozzle mm = 17.5, media mesh = 250).

For damage score, the most influential factors were media mesh (58.5%), nozzle mm (28.9%), pressure psi (12.5%). The best observed value was 1.5 (at pressure psi = 20, nozzle mm = 30, media mesh = 250).

Recommended Next Steps

  • Run confirmation experiments at the predicted optimal settings to validate the model.
  • Consider whether any fixed factors should be varied in a future study.

Step-by-Step Workflow

1

Preview the design

Terminal
$ python doe.py info --config use_cases/237_fossil_preparation/config.json
2

Generate the runner script

Terminal
$ python doe.py generate --config use_cases/237_fossil_preparation/config.json \ --output use_cases/237_fossil_preparation/results/run.sh --seed 42
3

Execute the experiments

Terminal
$ bash use_cases/237_fossil_preparation/results/run.sh
4

Analyze results

Terminal
$ python doe.py analyze --config use_cases/237_fossil_preparation/config.json
5

Get optimization recommendations

Terminal
$ python doe.py optimize --config use_cases/237_fossil_preparation/config.json
6

Multi-objective optimization

With 2 competing responses, use --multi to find the best compromise via Derringer–Suich desirability.

Terminal
$ python doe.py optimize --config use_cases/237_fossil_preparation/config.json --multi
7

Generate the HTML report

Terminal
$ python doe.py report --config use_cases/237_fossil_preparation/config.json \ --output use_cases/237_fossil_preparation/results/report.html
