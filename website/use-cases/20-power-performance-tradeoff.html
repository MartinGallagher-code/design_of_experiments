Central Composite

Power Capping & Performance Trade-off

Map the power-performance Pareto frontier on GPU-accelerated HPC nodes.

Summary

This experiment investigates power capping & performance trade-off. Central composite design to map the power-performance Pareto frontier on GPU-accelerated HPC nodes.

The design varies 3 factors: cpu power cap (watts), ranging from 120 to 240, gpu power cap (watts), ranging from 200 to 400, and mem freq (MHz), ranging from 2400 to 3200. The goal is to optimize 2 responses: gflops (GFLOPS) (maximize) and gflops per watt (GFLOPS/W) (maximize). Fixed conditions held constant across all runs include cpu model = EPYC_9654, gpu model = H100_SXM, nodes = 4, workload = HPL.

A Central Composite Design (CCD) was selected to fit a full quadratic response surface model, including curvature and interaction effects. With 3 factors this produces 22 runs including center points and axial (star) points that extend beyond the factorial range.

Quadratic response surface models were fitted to capture potential curvature and factor interactions. The RSM contour plots below visualize how pairs of factors jointly affect each response.

Key Findings

For gflops, the most influential factors were mem freq (42.6%), gpu power cap (38.8%), cpu power cap (18.5%). The best observed value was 36207.2146 (at cpu power cap = 180, gpu power cap = 300, mem freq = 2800).

For gflops per watt, the most influential factors were gpu power cap (54.6%), mem freq (27.2%), cpu power cap (18.2%). The best observed value was 6.43 (at cpu power cap = 180, gpu power cap = 300, mem freq = 2800).

Recommended Next Steps

  • Run confirmation experiments at the predicted optimal settings to validate the model.
  • Consider whether any fixed factors should be varied in a future study.
