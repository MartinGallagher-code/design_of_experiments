Rain Garden Design

Central composite design to maximize stormwater infiltration and minimize ponding time by tuning garden area, soil depth, and berm height

Summary

This experiment investigates rain garden design. Central composite design to maximize stormwater infiltration and minimize ponding time by tuning garden area, soil depth, and berm height.

The design varies 3 factors: area m2 (m2), ranging from 5 to 20, soil depth cm (cm), ranging from 30 to 90, and berm height cm (cm), ranging from 10 to 30. The goal is to optimize 2 responses: infiltration pct (%) (maximize) and ponding hrs (hrs) (minimize). Fixed conditions held constant across all runs include drainage area m2 = 200, soil type = sandy_loam.

A Central Composite Design (CCD) was selected to fit a full quadratic response surface model, including curvature and interaction effects. With 3 factors this produces 22 runs including center points and axial (star) points that extend beyond the factorial range.

Quadratic response surface models were fitted to capture potential curvature and factor interactions. The RSM contour plots below visualize how pairs of factors jointly affect each response.

Key Findings

For infiltration pct, the most influential factors were area m2 (53.4%), berm height cm (26.3%), soil depth cm (20.3%). The best observed value was 94.0 (at area m2 = 12.5, soil depth cm = 60, berm height cm = 1.74258).

For ponding hrs, the most influential factors were soil depth cm (50.5%), area m2 (27.7%), berm height cm (21.8%). The best observed value was 10.7 (at area m2 = 5, soil depth cm = 30, berm height cm = 30).

Recommended Next Steps

  • Run confirmation experiments at the predicted optimal settings to validate the model.
  • Consider whether any fixed factors should be varied in a future study.

Step-by-Step Workflow

1

Preview the design

Terminal
$ python doe.py info --config use_cases/136_rain_garden_sizing/config.json
2

Generate the runner script

Terminal
$ python doe.py generate --config use_cases/136_rain_garden_sizing/config.json \ --output use_cases/136_rain_garden_sizing/results/run.sh --seed 42
3

Execute the experiments

Terminal
$ bash use_cases/136_rain_garden_sizing/results/run.sh
4

Analyze results

Terminal
$ python doe.py analyze --config use_cases/136_rain_garden_sizing/config.json
5

Get optimization recommendations

Terminal
$ python doe.py optimize --config use_cases/136_rain_garden_sizing/config.json
6

Multi-objective optimization

With 2 competing responses, use --multi to find the best compromise via Derringer–Suich desirability.

Terminal
$ python doe.py optimize --config use_cases/136_rain_garden_sizing/config.json --multi
7

Generate the HTML report

Terminal
$ python doe.py report --config use_cases/136_rain_garden_sizing/config.json \ --output use_cases/136_rain_garden_sizing/results/report.html
