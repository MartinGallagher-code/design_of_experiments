Box-Behnken Design

Bread Baking Optimization

Box-Behnken design to optimize crust color, crumb texture, and rise height by tuning oven temperature, hydration, and proofing time

Summary

This experiment investigates bread baking optimization. Box-Behnken design to optimize crust color, crumb texture, and rise height by tuning oven temperature, hydration, and proofing time.

The design varies 3 factors: oven temp (C), ranging from 200 to 260, hydration pct (%), ranging from 60 to 80, and proof time (min), ranging from 30 to 120. The goal is to optimize 2 responses: crust score (pts) (maximize) and crumb score (pts) (maximize). Fixed conditions held constant across all runs include flour type = bread_flour, salt pct = 2.

A Box-Behnken design was chosen because it efficiently fits quadratic models with 3 continuous factors while avoiding extreme corner combinations — requiring only 15 runs instead of the 8 needed for a full factorial at two levels.

Quadratic response surface models were fitted to capture potential curvature and factor interactions. The RSM contour plots below visualize how pairs of factors jointly affect each response.

Key Findings

For crust score, the most influential factors were oven temp (42.8%), proof time (29.1%), hydration pct (28.1%). The best observed value was 7.7 (at oven temp = 230, hydration pct = 70, proof time = 75).

For crumb score, the most influential factors were oven temp (55.3%), hydration pct (25.0%), proof time (19.7%). The best observed value was 8.9 (at oven temp = 260, hydration pct = 60, proof time = 75).

Recommended Next Steps

  • Run confirmation experiments at the predicted optimal settings to validate the model.
  • Consider whether any fixed factors should be varied in a future study.

Step-by-Step Workflow

1

Preview the design

Terminal
$ python doe.py info --config use_cases/87_bread_baking/config.json
2

Generate the runner script

Terminal
$ python doe.py generate --config use_cases/87_bread_baking/config.json \ --output use_cases/87_bread_baking/results/run.sh --seed 42
3

Execute the experiments

Terminal
$ bash use_cases/87_bread_baking/results/run.sh
4

Analyze results

Terminal
$ python doe.py analyze --config use_cases/87_bread_baking/config.json
5

Get optimization recommendations

Terminal
$ python doe.py optimize --config use_cases/87_bread_baking/config.json
6

Multi-objective optimization

With 2 competing responses, use --multi to find the best compromise via Derringer–Suich desirability.

Terminal
$ python doe.py optimize --config use_cases/87_bread_baking/config.json --multi
7

Generate the HTML report

Terminal
$ python doe.py report --config use_cases/87_bread_baking/config.json \ --output use_cases/87_bread_baking/results/report.html
