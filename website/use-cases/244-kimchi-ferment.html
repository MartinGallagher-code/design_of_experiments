Central Composite Design

Kimchi Fermentation Timing

Central composite design to optimize tang level and texture by tuning salt brine strength, fermentation temperature, and gochugaru amount

Summary

This experiment investigates kimchi fermentation timing. Central composite design to optimize tang level and texture by tuning salt brine strength, fermentation temperature, and gochugaru amount.

The design varies 3 factors: brine pct (%), ranging from 3 to 8, ferm temp c (C), ranging from 2 to 22, and gochugaru pct (%), ranging from 3 to 10. The goal is to optimize 2 responses: tang level (pts) (maximize) and texture score (pts) (maximize). Fixed conditions held constant across all runs include cabbage type = napa, garlic pct = 3.

A Central Composite Design (CCD) was selected to fit a full quadratic response surface model, including curvature and interaction effects. With 3 factors this produces 22 runs including center points and axial (star) points that extend beyond the factorial range.

Quadratic response surface models were fitted to capture potential curvature and factor interactions. The RSM contour plots below visualize how pairs of factors jointly affect each response.

Key Findings

For tang level, the most influential factors were gochugaru pct (62.2%), brine pct (22.0%), ferm temp c (15.9%). The best observed value was 7.4 (at brine pct = 8, ferm temp c = 2, gochugaru pct = 10).

For texture score, the most influential factors were gochugaru pct (78.0%), brine pct (14.5%), ferm temp c (7.5%). The best observed value was 9.5 (at brine pct = 5.5, ferm temp c = -6.25742, gochugaru pct = 6.5).

Recommended Next Steps

  • Run confirmation experiments at the predicted optimal settings to validate the model.
  • Consider whether any fixed factors should be varied in a future study.

Step-by-Step Workflow

1

Preview the design

Terminal
$ python doe.py info --config use_cases/244_kimchi_ferment/config.json
2

Generate the runner script

Terminal
$ python doe.py generate --config use_cases/244_kimchi_ferment/config.json \ --output use_cases/244_kimchi_ferment/results/run.sh --seed 42
3

Execute the experiments

Terminal
$ bash use_cases/244_kimchi_ferment/results/run.sh
4

Analyze results

Terminal
$ python doe.py analyze --config use_cases/244_kimchi_ferment/config.json
5

Get optimization recommendations

Terminal
$ python doe.py optimize --config use_cases/244_kimchi_ferment/config.json
6

Multi-objective optimization

With 2 competing responses, use --multi to find the best compromise via Derringer–Suich desirability.

Terminal
$ python doe.py optimize --config use_cases/244_kimchi_ferment/config.json --multi
7

Generate the HTML report

Terminal
$ python doe.py report --config use_cases/244_kimchi_ferment/config.json \ --output use_cases/244_kimchi_ferment/results/report.html
