Box-Behnken Design

Deployment Canary Rollout

Box-Behnken design to tune canary percentage, evaluation window, and error threshold for rollout safety

Summary

This experiment investigates deployment canary rollout. Box-Behnken design to tune canary percentage, evaluation window, and error threshold for rollout safety.

The design varies 3 factors: canary pct (%), ranging from 5 to 25, evaluation window min (min), ranging from 5 to 30, and error threshold pct (%), ranging from 0.5 to 5.0. The goal is to optimize 2 responses: rollout safety score (score) (maximize) and deployment time min (min) (minimize). Fixed conditions held constant across all runs include orchestrator = kubernetes, strategy = canary.

A Box-Behnken design was chosen because it efficiently fits quadratic models with 3 continuous factors while avoiding extreme corner combinations — requiring only 15 runs instead of the 8 needed for a full factorial at two levels.

Quadratic response surface models were fitted to capture potential curvature and factor interactions. The RSM contour plots below visualize how pairs of factors jointly affect each response.

Key Findings

For rollout safety score, the most influential factors were error threshold pct (52.2%), canary pct (39.8%), evaluation window min (8.0%). The best observed value was 84.4 (at canary pct = 5, evaluation window min = 30, error threshold pct = 2.75).

For deployment time min, the most influential factors were error threshold pct (41.7%), canary pct (32.5%), evaluation window min (25.8%). The best observed value was 3.1 (at canary pct = 15, evaluation window min = 5, error threshold pct = 5).

Recommended Next Steps

  • Run confirmation experiments at the predicted optimal settings to validate the model.
  • Consider whether any fixed factors should be varied in a future study.

Step-by-Step Workflow

1

Preview the design

Terminal
$ python doe.py info --config use_cases/78_deployment_canary_rollout/config.json
2

Generate the runner script

Terminal
$ python doe.py generate --config use_cases/78_deployment_canary_rollout/config.json \ --output use_cases/78_deployment_canary_rollout/results/run.sh --seed 42
3

Execute the experiments

Terminal
$ bash use_cases/78_deployment_canary_rollout/results/run.sh
4

Analyze results

Terminal
$ python doe.py analyze --config use_cases/78_deployment_canary_rollout/config.json
5

Get optimization recommendations

Terminal
$ python doe.py optimize --config use_cases/78_deployment_canary_rollout/config.json
6

Multi-objective optimization

With 2 competing responses, use --multi to find the best compromise via Derringer–Suich desirability.

Terminal
$ python doe.py optimize --config use_cases/78_deployment_canary_rollout/config.json --multi
7

Generate the HTML report

Terminal
$ python doe.py report --config use_cases/78_deployment_canary_rollout/config.json \ --output use_cases/78_deployment_canary_rollout/results/report.html
