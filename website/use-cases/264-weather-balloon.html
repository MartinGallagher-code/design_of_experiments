Box-Behnken Design

Weather Balloon Launch Parameters

Box-Behnken design to maximize burst altitude and minimize payload swing by tuning helium fill volume, balloon mass, and payload weight

Summary

This experiment investigates weather balloon launch parameters. Box-Behnken design to maximize burst altitude and minimize payload swing by tuning helium fill volume, balloon mass, and payload weight.

The design varies 3 factors: fill m3 (m3), ranging from 0.8 to 2.0, balloon g (g), ranging from 300 to 1200, and payload g (g), ranging from 100 to 500. The goal is to optimize 2 responses: burst alt km (km) (maximize) and swing deg (deg) (minimize). Fixed conditions held constant across all runs include gas = helium, radiosonde = RS41.

A Box-Behnken design was chosen because it efficiently fits quadratic models with 3 continuous factors while avoiding extreme corner combinations — requiring only 15 runs instead of the 8 needed for a full factorial at two levels.

Quadratic response surface models were fitted to capture potential curvature and factor interactions. The RSM contour plots below visualize how pairs of factors jointly affect each response.

Key Findings

For burst alt km, the most influential factors were fill m3 (64.8%), payload g (26.2%), balloon g (9.0%). The best observed value was 34.2 (at fill m3 = 2, balloon g = 300, payload g = 300).

For swing deg, the most influential factors were payload g (42.9%), balloon g (39.1%), fill m3 (18.0%). The best observed value was 9.0 (at fill m3 = 1.4, balloon g = 1200, payload g = 100).

Recommended Next Steps

  • Run confirmation experiments at the predicted optimal settings to validate the model.
  • Consider whether any fixed factors should be varied in a future study.

Step-by-Step Workflow

1

Preview the design

Terminal
$ python doe.py info --config use_cases/264_weather_balloon/config.json
2

Generate the runner script

Terminal
$ python doe.py generate --config use_cases/264_weather_balloon/config.json \ --output use_cases/264_weather_balloon/results/run.sh --seed 42
3

Execute the experiments

Terminal
$ bash use_cases/264_weather_balloon/results/run.sh
4

Analyze results

Terminal
$ python doe.py analyze --config use_cases/264_weather_balloon/config.json
5

Get optimization recommendations

Terminal
$ python doe.py optimize --config use_cases/264_weather_balloon/config.json
6

Multi-objective optimization

With 2 competing responses, use --multi to find the best compromise via Derringer–Suich desirability.

Terminal
$ python doe.py optimize --config use_cases/264_weather_balloon/config.json --multi
7

Generate the HTML report

Terminal
$ python doe.py report --config use_cases/264_weather_balloon/config.json \ --output use_cases/264_weather_balloon/results/report.html
