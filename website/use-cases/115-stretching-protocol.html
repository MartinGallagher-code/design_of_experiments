Box-Behnken Design

Stretching Protocol Design

Box-Behnken design to maximize flexibility gain and minimize soreness by tuning hold duration, session frequency, and warmup intensity

Summary

This experiment investigates stretching protocol design. Box-Behnken design to maximize flexibility gain and minimize soreness by tuning hold duration, session frequency, and warmup intensity.

The design varies 3 factors: hold sec (sec), ranging from 10 to 60, sessions per week (per_week), ranging from 2 to 7, and warmup min (min), ranging from 0 to 15. The goal is to optimize 2 responses: flexibility gain (cm) (maximize) and soreness (pts) (minimize). Fixed conditions held constant across all runs include stretch type = static, target area = hamstrings.

A Box-Behnken design was chosen because it efficiently fits quadratic models with 3 continuous factors while avoiding extreme corner combinations — requiring only 15 runs instead of the 8 needed for a full factorial at two levels.

Quadratic response surface models were fitted to capture potential curvature and factor interactions. The RSM contour plots below visualize how pairs of factors jointly affect each response.

Key Findings

For flexibility gain, the most influential factors were sessions per week (39.5%), hold sec (33.8%), warmup min (26.7%). The best observed value was 5.8 (at hold sec = 60, sessions per week = 2, warmup min = 7.5).

For soreness, the most influential factors were sessions per week (64.7%), warmup min (24.9%), hold sec (10.4%). The best observed value was 2.1 (at hold sec = 10, sessions per week = 4.5, warmup min = 15).

Recommended Next Steps

  • Run confirmation experiments at the predicted optimal settings to validate the model.
  • Consider whether any fixed factors should be varied in a future study.

Step-by-Step Workflow

1

Preview the design

Terminal
$ python doe.py info --config use_cases/115_stretching_protocol/config.json
2

Generate the runner script

Terminal
$ python doe.py generate --config use_cases/115_stretching_protocol/config.json \ --output use_cases/115_stretching_protocol/results/run.sh --seed 42
3

Execute the experiments

Terminal
$ bash use_cases/115_stretching_protocol/results/run.sh
4

Analyze results

Terminal
$ python doe.py analyze --config use_cases/115_stretching_protocol/config.json
5

Get optimization recommendations

Terminal
$ python doe.py optimize --config use_cases/115_stretching_protocol/config.json
6

Multi-objective optimization

With 2 competing responses, use --multi to find the best compromise via Derringer–Suich desirability.

Terminal
$ python doe.py optimize --config use_cases/115_stretching_protocol/config.json --multi
7

Generate the HTML report

Terminal
$ python doe.py report --config use_cases/115_stretching_protocol/config.json \ --output use_cases/115_stretching_protocol/results/report.html
