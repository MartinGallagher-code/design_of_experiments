Box-Behnken Design

BLE Mesh Topology

Box-Behnken design to tune relay count, TTL hops, and publish interval for message delivery and latency

Summary

This experiment investigates ble mesh topology. Box-Behnken design to tune relay count, TTL hops, and publish interval for message delivery and latency.

The design varies 3 factors: relay count (nodes), ranging from 2 to 10, ttl hops (hops), ranging from 2 to 8, and publish interval ms (ms), ranging from 100 to 2000. The goal is to optimize 2 responses: message delivery pct (%) (maximize) and network latency ms (ms) (minimize). Fixed conditions held constant across all runs include ble version = 5.0, mesh profile = sig_mesh.

A Box-Behnken design was chosen because it efficiently fits quadratic models with 3 continuous factors while avoiding extreme corner combinations — requiring only 15 runs instead of the 8 needed for a full factorial at two levels.

Quadratic response surface models were fitted to capture potential curvature and factor interactions. The RSM contour plots below visualize how pairs of factors jointly affect each response.

Key Findings

For message delivery pct, the most influential factors were relay count (41.9%), publish interval ms (29.8%), ttl hops (28.3%). The best observed value was 97.6 (at relay count = 6, ttl hops = 5, publish interval ms = 1050).

For network latency ms, the most influential factors were ttl hops (49.9%), relay count (40.1%), publish interval ms (10.0%). The best observed value was 31.0 (at relay count = 6, ttl hops = 5, publish interval ms = 1050).

Recommended Next Steps

  • Run confirmation experiments at the predicted optimal settings to validate the model.
  • Consider whether any fixed factors should be varied in a future study.

Step-by-Step Workflow

1

Preview the design

Terminal
$ python doe.py info --config use_cases/68_ble_mesh_topology/config.json
2

Generate the runner script

Terminal
$ python doe.py generate --config use_cases/68_ble_mesh_topology/config.json \ --output use_cases/68_ble_mesh_topology/results/run.sh --seed 42
3

Execute the experiments

Terminal
$ bash use_cases/68_ble_mesh_topology/results/run.sh
4

Analyze results

Terminal
$ python doe.py analyze --config use_cases/68_ble_mesh_topology/config.json
5

Get optimization recommendations

Terminal
$ python doe.py optimize --config use_cases/68_ble_mesh_topology/config.json
6

Multi-objective optimization

With 2 competing responses, use --multi to find the best compromise via Derringer–Suich desirability.

Terminal
$ python doe.py optimize --config use_cases/68_ble_mesh_topology/config.json --multi
7

Generate the HTML report

Terminal
$ python doe.py report --config use_cases/68_ble_mesh_topology/config.json \ --output use_cases/68_ble_mesh_topology/results/report.html
