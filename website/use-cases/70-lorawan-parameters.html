Central Composite Design

LoRaWAN Parameters

Central Composite design to optimize spreading factor, TX power, and coding rate for range and battery life

Summary

This experiment investigates lorawan parameters. Central Composite design to optimize spreading factor, TX power, and coding rate for range and battery life.

The design varies 3 factors: spreading factor (SF), ranging from 7 to 12, tx power dbm (dBm), ranging from 2 to 20, and coding rate (CR), ranging from 5 to 8. The goal is to optimize 2 responses: range km (km) (maximize) and battery life days (days) (maximize). Fixed conditions held constant across all runs include frequency = 915MHz, bandwidth = 125kHz.

A Central Composite Design (CCD) was selected to fit a full quadratic response surface model, including curvature and interaction effects. With 3 factors this produces 22 runs including center points and axial (star) points that extend beyond the factorial range.

Quadratic response surface models were fitted to capture potential curvature and factor interactions. The RSM contour plots below visualize how pairs of factors jointly affect each response.

Key Findings

For range km, the most influential factors were spreading factor (39.0%), tx power dbm (34.5%), coding rate (26.5%). The best observed value was 10.9 (at spreading factor = 14.0644, tx power dbm = 11, coding rate = 6.5).

For battery life days, the most influential factors were tx power dbm (37.3%), spreading factor (36.2%), coding rate (26.5%). The best observed value was 554.0 (at spreading factor = 7, tx power dbm = 20, coding rate = 8).

Recommended Next Steps

  • Run confirmation experiments at the predicted optimal settings to validate the model.
  • Consider whether any fixed factors should be varied in a future study.

Step-by-Step Workflow

1

Preview the design

Terminal
$ python doe.py info --config use_cases/70_lorawan_parameters/config.json
2

Generate the runner script

Terminal
$ python doe.py generate --config use_cases/70_lorawan_parameters/config.json \ --output use_cases/70_lorawan_parameters/results/run.sh --seed 42
3

Execute the experiments

Terminal
$ bash use_cases/70_lorawan_parameters/results/run.sh
4

Analyze results

Terminal
$ python doe.py analyze --config use_cases/70_lorawan_parameters/config.json
5

Get optimization recommendations

Terminal
$ python doe.py optimize --config use_cases/70_lorawan_parameters/config.json
6

Multi-objective optimization

With 2 competing responses, use --multi to find the best compromise via Derringer–Suich desirability.

Terminal
$ python doe.py optimize --config use_cases/70_lorawan_parameters/config.json --multi
7

Generate the HTML report

Terminal
$ python doe.py report --config use_cases/70_lorawan_parameters/config.json \ --output use_cases/70_lorawan_parameters/results/report.html
