Box-Behnken Design

Rainwater Harvesting System

Box-Behnken design to maximize water captured and minimize overflow by tuning tank size, gutter area, and first-flush diverter volume

Summary

This experiment investigates rainwater harvesting system. Box-Behnken design to maximize water captured and minimize overflow by tuning tank size, gutter area, and first-flush diverter volume.

The design varies 3 factors: tank liters (L), ranging from 500 to 5000, gutter area m2 (m2), ranging from 50 to 200, and first flush L (L), ranging from 10 to 80. The goal is to optimize 2 responses: capture pct (%) (maximize) and overflow pct (%) (minimize). Fixed conditions held constant across all runs include annual rainfall mm = 900, usage L day = 50.

A Box-Behnken design was chosen because it efficiently fits quadratic models with 3 continuous factors while avoiding extreme corner combinations — requiring only 15 runs instead of the 8 needed for a full factorial at two levels.

Quadratic response surface models were fitted to capture potential curvature and factor interactions. The RSM contour plots below visualize how pairs of factors jointly affect each response.

Key Findings

For capture pct, the most influential factors were gutter area m2 (47.5%), tank liters (31.0%), first flush L (21.5%). The best observed value was 73.6 (at tank liters = 2750, gutter area m2 = 50, first flush L = 80).

For overflow pct, the most influential factors were gutter area m2 (48.6%), tank liters (36.2%), first flush L (15.2%). The best observed value was 17.1 (at tank liters = 2750, gutter area m2 = 50, first flush L = 80).

Recommended Next Steps

  • Run confirmation experiments at the predicted optimal settings to validate the model.
  • Consider whether any fixed factors should be varied in a future study.

Step-by-Step Workflow

1

Preview the design

Terminal
$ python doe.py info --config use_cases/128_rainwater_harvesting/config.json
2

Generate the runner script

Terminal
$ python doe.py generate --config use_cases/128_rainwater_harvesting/config.json \ --output use_cases/128_rainwater_harvesting/results/run.sh --seed 42
3

Execute the experiments

Terminal
$ bash use_cases/128_rainwater_harvesting/results/run.sh
4

Analyze results

Terminal
$ python doe.py analyze --config use_cases/128_rainwater_harvesting/config.json
5

Get optimization recommendations

Terminal
$ python doe.py optimize --config use_cases/128_rainwater_harvesting/config.json
6

Multi-objective optimization

With 2 competing responses, use --multi to find the best compromise via Derringer–Suich desirability.

Terminal
$ python doe.py optimize --config use_cases/128_rainwater_harvesting/config.json --multi
7

Generate the HTML report

Terminal
$ python doe.py report --config use_cases/128_rainwater_harvesting/config.json \ --output use_cases/128_rainwater_harvesting/results/report.html
