Ginger Beer Fermentation

Central composite design to maximize ginger bite and carbonation while controlling sweetness by tuning ginger amount, sugar, and fermentation time

Summary

This experiment investigates ginger beer fermentation. Central composite design to maximize ginger bite and carbonation while controlling sweetness by tuning ginger amount, sugar, and fermentation time.

The design varies 3 factors: ginger g L (g/L), ranging from 20 to 80, sugar g L (g/L), ranging from 40 to 100, and ferm days (days), ranging from 2 to 7. The goal is to optimize 2 responses: ginger bite (pts) (maximize) and carbonation (pts) (maximize). Fixed conditions held constant across all runs include starter = ginger_bug, lemon juice = 15mL/L.

A Central Composite Design (CCD) was selected to fit a full quadratic response surface model, including curvature and interaction effects. With 3 factors this produces 22 runs including center points and axial (star) points that extend beyond the factorial range.

Quadratic response surface models were fitted to capture potential curvature and factor interactions. The RSM contour plots below visualize how pairs of factors jointly affect each response.

Key Findings

For ginger bite, the most influential factors were ginger g L (43.2%), ferm days (31.2%), sugar g L (25.7%). The best observed value was 7.7 (at ginger g L = 50, sugar g L = 70, ferm days = 4.5).

For carbonation, the most influential factors were ferm days (56.2%), ginger g L (32.5%), sugar g L (11.3%). The best observed value was 7.1 (at ginger g L = 50, sugar g L = 70, ferm days = 4.5).

Recommended Next Steps

  • Run confirmation experiments at the predicted optimal settings to validate the model.
  • Consider whether any fixed factors should be varied in a future study.

Step-by-Step Workflow

1

Preview the design

Terminal
$ python doe.py info --config use_cases/248_ginger_beer/config.json
2

Generate the runner script

Terminal
$ python doe.py generate --config use_cases/248_ginger_beer/config.json \ --output use_cases/248_ginger_beer/results/run.sh --seed 42
3

Execute the experiments

Terminal
$ bash use_cases/248_ginger_beer/results/run.sh
4

Analyze results

Terminal
$ python doe.py analyze --config use_cases/248_ginger_beer/config.json
5

Get optimization recommendations

Terminal
$ python doe.py optimize --config use_cases/248_ginger_beer/config.json
6

Multi-objective optimization

With 2 competing responses, use --multi to find the best compromise via Derringer–Suich desirability.

Terminal
$ python doe.py optimize --config use_cases/248_ginger_beer/config.json --multi
7

Generate the HTML report

Terminal
$ python doe.py report --config use_cases/248_ginger_beer/config.json \ --output use_cases/248_ginger_beer/results/report.html
