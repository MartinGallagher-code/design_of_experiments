Box-Behnken Design

Groundwater Sampling Protocol

Box-Behnken design to maximize sample representativeness and minimize turbidity by tuning purge volume, pump rate, and well development time

Summary

This experiment investigates groundwater sampling protocol. Box-Behnken design to maximize sample representativeness and minimize turbidity by tuning purge volume, pump rate, and well development time.

The design varies 3 factors: purge volumes (well_vols), ranging from 1 to 5, pump rate lpm (L/min), ranging from 0.5 to 5.0, and dev hrs (hrs), ranging from 1 to 8. The goal is to optimize 2 responses: representativeness (pts) (maximize) and sample turbidity (NTU) (minimize). Fixed conditions held constant across all runs include well type = monitoring, analytes = metals.

A Box-Behnken design was chosen because it efficiently fits quadratic models with 3 continuous factors while avoiding extreme corner combinations — requiring only 15 runs instead of the 8 needed for a full factorial at two levels.

Quadratic response surface models were fitted to capture potential curvature and factor interactions. The RSM contour plots below visualize how pairs of factors jointly affect each response.

Key Findings

For representativeness, the most influential factors were dev hrs (44.5%), purge volumes (32.5%), pump rate lpm (22.9%). The best observed value was 8.0 (at purge volumes = 1, pump rate lpm = 2.75, dev hrs = 1).

For sample turbidity, the most influential factors were dev hrs (51.7%), purge volumes (46.4%), pump rate lpm (1.8%). The best observed value was 4.7 (at purge volumes = 3, pump rate lpm = 2.75, dev hrs = 4.5).

Recommended Next Steps

  • Run confirmation experiments at the predicted optimal settings to validate the model.
  • Consider whether any fixed factors should be varied in a future study.

Step-by-Step Workflow

1

Preview the design

Terminal
$ python doe.py info --config use_cases/235_groundwater_sampling/config.json
2

Generate the runner script

Terminal
$ python doe.py generate --config use_cases/235_groundwater_sampling/config.json \ --output use_cases/235_groundwater_sampling/results/run.sh --seed 42
3

Execute the experiments

Terminal
$ bash use_cases/235_groundwater_sampling/results/run.sh
4

Analyze results

Terminal
$ python doe.py analyze --config use_cases/235_groundwater_sampling/config.json
5

Get optimization recommendations

Terminal
$ python doe.py optimize --config use_cases/235_groundwater_sampling/config.json
6

Multi-objective optimization

With 2 competing responses, use --multi to find the best compromise via Derringer–Suich desirability.

Terminal
$ python doe.py optimize --config use_cases/235_groundwater_sampling/config.json --multi
7

Generate the HTML report

Terminal
$ python doe.py report --config use_cases/235_groundwater_sampling/config.json \ --output use_cases/235_groundwater_sampling/results/report.html
