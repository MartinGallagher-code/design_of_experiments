Central Composite Design

Bicycle Gearing & Cadence

Central composite design to maximize average speed and minimize perceived effort by tuning gear ratio, cadence, and tire pressure

Summary

This experiment investigates bicycle gearing & cadence. Central composite design to maximize average speed and minimize perceived effort by tuning gear ratio, cadence, and tire pressure.

The design varies 3 factors: gear ratio (ratio), ranging from 2.0 to 4.0, cadence rpm (rpm), ranging from 60 to 100, and tire psi (psi), ranging from 60 to 110. The goal is to optimize 2 responses: avg speed kph (kph) (maximize) and effort score (pts) (minimize). Fixed conditions held constant across all runs include terrain = rolling_hills, rider weight = 75kg.

A Central Composite Design (CCD) was selected to fit a full quadratic response surface model, including curvature and interaction effects. With 3 factors this produces 22 runs including center points and axial (star) points that extend beyond the factorial range.

Quadratic response surface models were fitted to capture potential curvature and factor interactions. The RSM contour plots below visualize how pairs of factors jointly affect each response.

Key Findings

For avg speed kph, the most influential factors were tire psi (52.6%), cadence rpm (30.5%), gear ratio (16.9%). The best observed value was 26.9 (at gear ratio = 3, cadence rpm = 80, tire psi = 85).

For effort score, the most influential factors were gear ratio (41.7%), cadence rpm (31.2%), tire psi (27.1%). The best observed value was 4.2 (at gear ratio = 4, cadence rpm = 60, tire psi = 110).

Recommended Next Steps

  • Run confirmation experiments at the predicted optimal settings to validate the model.
  • Consider whether any fixed factors should be varied in a future study.

Step-by-Step Workflow

1

Preview the design

Terminal
$ python doe.py info --config use_cases/121_bicycle_gear_cadence/config.json
2

Generate the runner script

Terminal
$ python doe.py generate --config use_cases/121_bicycle_gear_cadence/config.json \ --output use_cases/121_bicycle_gear_cadence/results/run.sh --seed 42
3

Execute the experiments

Terminal
$ bash use_cases/121_bicycle_gear_cadence/results/run.sh
4

Analyze results

Terminal
$ python doe.py analyze --config use_cases/121_bicycle_gear_cadence/config.json
5

Get optimization recommendations

Terminal
$ python doe.py optimize --config use_cases/121_bicycle_gear_cadence/config.json
6

Multi-objective optimization

With 2 competing responses, use --multi to find the best compromise via Derringer–Suich desirability.

Terminal
$ python doe.py optimize --config use_cases/121_bicycle_gear_cadence/config.json --multi
7

Generate the HTML report

Terminal
$ python doe.py report --config use_cases/121_bicycle_gear_cadence/config.json \ --output use_cases/121_bicycle_gear_cadence/results/report.html
