Central Composite Design

Video Encoding Quality

Central composite design to maximize visual quality and minimize file size by tuning bitrate, CRF value, and GOP length

Summary

This experiment investigates video encoding quality. Central composite design to maximize visual quality and minimize file size by tuning bitrate, CRF value, and GOP length.

The design varies 3 factors: bitrate mbps (Mbps), ranging from 5 to 50, crf (CRF), ranging from 18 to 28, and gop frames (frames), ranging from 15 to 120. The goal is to optimize 2 responses: vmaf score (VMAF) (maximize) and file size mb (MB/min) (minimize). Fixed conditions held constant across all runs include codec = h265, resolution = 4K.

A Central Composite Design (CCD) was selected to fit a full quadratic response surface model, including curvature and interaction effects. With 3 factors this produces 22 runs including center points and axial (star) points that extend beyond the factorial range.

Quadratic response surface models were fitted to capture potential curvature and factor interactions. The RSM contour plots below visualize how pairs of factors jointly affect each response.

Key Findings

For vmaf score, the most influential factors were gop frames (54.3%), crf (30.5%), bitrate mbps (15.2%). The best observed value was 92.0 (at bitrate mbps = 50, crf = 18, gop frames = 120).

For file size mb, the most influential factors were gop frames (34.8%), crf (33.8%), bitrate mbps (31.4%). The best observed value was 48.0 (at bitrate mbps = 5, crf = 18, gop frames = 120).

Recommended Next Steps

  • Run confirmation experiments at the predicted optimal settings to validate the model.
  • Consider whether any fixed factors should be varied in a future study.

Step-by-Step Workflow

1

Preview the design

Terminal
$ python doe.py info --config use_cases/156_video_encoding/config.json
2

Generate the runner script

Terminal
$ python doe.py generate --config use_cases/156_video_encoding/config.json \ --output use_cases/156_video_encoding/results/run.sh --seed 42
3

Execute the experiments

Terminal
$ bash use_cases/156_video_encoding/results/run.sh
4

Analyze results

Terminal
$ python doe.py analyze --config use_cases/156_video_encoding/config.json
5

Get optimization recommendations

Terminal
$ python doe.py optimize --config use_cases/156_video_encoding/config.json
6

Multi-objective optimization

With 2 competing responses, use --multi to find the best compromise via Derringer–Suich desirability.

Terminal
$ python doe.py optimize --config use_cases/156_video_encoding/config.json --multi
7

Generate the HTML report

Terminal
$ python doe.py report --config use_cases/156_video_encoding/config.json \ --output use_cases/156_video_encoding/results/report.html
