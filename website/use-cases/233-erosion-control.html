Box-Behnken Design

Hillside Erosion Control

Box-Behnken design to minimize soil loss and maximize vegetation establishment by tuning mulch depth, seed mix density, and terracing interval

Summary

This experiment investigates hillside erosion control. Box-Behnken design to minimize soil loss and maximize vegetation establishment by tuning mulch depth, seed mix density, and terracing interval.

The design varies 3 factors: mulch cm (cm), ranging from 2 to 10, seed g m2 (g/m2), ranging from 10 to 50, and terrace m (m), ranging from 5 to 20. The goal is to optimize 2 responses: soil loss t ha (t/ha/yr) (minimize) and vegetation pct (%) (maximize). Fixed conditions held constant across all runs include slope pct = 30, soil type = silt_loam.

A Box-Behnken design was chosen because it efficiently fits quadratic models with 3 continuous factors while avoiding extreme corner combinations — requiring only 15 runs instead of the 8 needed for a full factorial at two levels.

Quadratic response surface models were fitted to capture potential curvature and factor interactions. The RSM contour plots below visualize how pairs of factors jointly affect each response.

Key Findings

For soil loss t ha, the most influential factors were terrace m (38.2%), seed g m2 (36.7%), mulch cm (25.1%). The best observed value was 7.6 (at mulch cm = 6, seed g m2 = 30, terrace m = 12.5).

For vegetation pct, the most influential factors were seed g m2 (37.9%), mulch cm (33.8%), terrace m (28.3%). The best observed value was 75.0 (at mulch cm = 10, seed g m2 = 30, terrace m = 5).

Recommended Next Steps

  • Run confirmation experiments at the predicted optimal settings to validate the model.
  • Consider whether any fixed factors should be varied in a future study.

Step-by-Step Workflow

1

Preview the design

Terminal
$ python doe.py info --config use_cases/233_erosion_control/config.json
2

Generate the runner script

Terminal
$ python doe.py generate --config use_cases/233_erosion_control/config.json \ --output use_cases/233_erosion_control/results/run.sh --seed 42
3

Execute the experiments

Terminal
$ bash use_cases/233_erosion_control/results/run.sh
4

Analyze results

Terminal
$ python doe.py analyze --config use_cases/233_erosion_control/config.json
5

Get optimization recommendations

Terminal
$ python doe.py optimize --config use_cases/233_erosion_control/config.json
6

Multi-objective optimization

With 2 competing responses, use --multi to find the best compromise via Derringer–Suich desirability.

Terminal
$ python doe.py optimize --config use_cases/233_erosion_control/config.json --multi
7

Generate the HTML report

Terminal
$ python doe.py report --config use_cases/233_erosion_control/config.json \ --output use_cases/233_erosion_control/results/report.html
