Yoga Sequence Design

Box-Behnken design to maximize flexibility gain and relaxation by tuning hold duration, transition speed, and warmup length

Summary

This experiment investigates yoga sequence design. Box-Behnken design to maximize flexibility gain and relaxation by tuning hold duration, transition speed, and warmup length.

The design varies 3 factors: hold sec (sec), ranging from 15 to 60, transition pace (level), ranging from 1 to 5, and warmup min (min), ranging from 5 to 20. The goal is to optimize 2 responses: flexibility gain (pts) (maximize) and relaxation score (pts) (maximize). Fixed conditions held constant across all runs include session length = 60min, level = intermediate.

A Box-Behnken design was chosen because it efficiently fits quadratic models with 3 continuous factors while avoiding extreme corner combinations — requiring only 15 runs instead of the 8 needed for a full factorial at two levels.

Quadratic response surface models were fitted to capture potential curvature and factor interactions. The RSM contour plots below visualize how pairs of factors jointly affect each response.

Key Findings

For flexibility gain, the most influential factors were transition pace (41.0%), warmup min (30.3%), hold sec (28.7%). The best observed value was 7.4 (at hold sec = 15, transition pace = 5, warmup min = 12.5).

For relaxation score, the most influential factors were hold sec (56.0%), transition pace (32.6%), warmup min (11.4%). The best observed value was 8.2 (at hold sec = 15, transition pace = 3, warmup min = 20).

Recommended Next Steps

  • Run confirmation experiments at the predicted optimal settings to validate the model.
  • Consider whether any fixed factors should be varied in a future study.

Step-by-Step Workflow

1

Preview the design

Terminal
$ python doe.py info --config use_cases/217_yoga_sequence/config.json
2

Generate the runner script

Terminal
$ python doe.py generate --config use_cases/217_yoga_sequence/config.json \ --output use_cases/217_yoga_sequence/results/run.sh --seed 42
3

Execute the experiments

Terminal
$ bash use_cases/217_yoga_sequence/results/run.sh
4

Analyze results

Terminal
$ python doe.py analyze --config use_cases/217_yoga_sequence/config.json
5

Get optimization recommendations

Terminal
$ python doe.py optimize --config use_cases/217_yoga_sequence/config.json
6

Multi-objective optimization

With 2 competing responses, use --multi to find the best compromise via Derringer–Suich desirability.

Terminal
$ python doe.py optimize --config use_cases/217_yoga_sequence/config.json --multi
7

Generate the HTML report

Terminal
$ python doe.py report --config use_cases/217_yoga_sequence/config.json \ --output use_cases/217_yoga_sequence/results/report.html
